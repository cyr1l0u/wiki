# 必备

- UML
- 设计模式
- SOLID 原则
- 坏味道（Code Smell）
- 测试驱动开发（TDD）
- 版本控制
- 模块化和分层
- 代码格式化
- 持续集成和持续交付

## 详细

- SOLID 原则：
  - 单一职责原则
  - 开闭原则
  - 里氏代换原则
  - 依赖倒转原则
  - 接口隔离原则
  - 合成复用原则
  - 迪米特法则
- 坏味道（Code Smell）
  - 代码重复
  - 过大的函数 / 类 / 模块
  - 不恰当的命名
  - 不一致的风格
  - 不合理的复杂性
  - 过多的 if else
  - 过多的参数
  - 紧密耦合
  - 不恰当的注释或没有注释
